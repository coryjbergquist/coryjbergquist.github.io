The Floor Is Java

10 Feb 2019

Look out the floor is Lava!

Image from https://gamerant.com/floor-is-lava-dont-starve/

When you were a kid you probably played the game where you pretend the floor is lava and you have to jump from couches onto pillows and blankets to move from room to room. Well, as I dove deeper into my job search as a web developer I found myself jumping from job posting to job posting pretending like I should only look for jobs with my specifically learned languages.

It’s no secret that Java is part of a whole lot of development jobs out there. Not pursuing a position simply because it’s not your main focus can seriously hinder your job search, especially if you are just getting started. As a developer, we are masters of learning the latest and greatest technologies, so what is stopping us from taking some time to learn one of the tried and true languages of the coding world?

After a few weeks of reading through introductions to Java and looking at a lot of code, I quickly realized there are plenty of similarities to the languages I was familiar with (like Javascript and Ruby), and being able to at least understand the main points of Java could be a huge bonus while I continue my job search.

Having said that, I’m going to go over some of the very basics of Java, point out some similarities to Ruby and Javascript, and hopefully get the ball rolling for some of you who might be a little hesitant to start learning Java.

Let’s start with printing some text:

public class YourName {
	public static void main(String[] args) {

		System.out.println("My name is Obi Wan Kenobi");

	}
}

The important line here is:

  System.out.println("My name is Obi Wan Kenobi");

For those familiar with Ruby, this would be similar to puts "My name is...", or rendering text to the DOM in Javascript by calling a function that includes return "My name is..."

Data types!

A coding language should have data types right? Java certainly does. A few of the main ones shouldn’t be a surprise to anyone familiar with other coding languages.

The int data type is for integers (32 bit), the char data type holds only one character such as “A”, and the boolean can only be true or false. int and boolean are similar to what you’d expect from Ruby or Javascript, however the char data type might be a new one for you. It’s specifically meant to hold one character, so if you have a variable that will always hold one character at time, you could use char.

But what about Strings?

Strings aren’t a data type in Java. A string will be an instance of the String class and can be created like so:

String help = new String ("Help me, Obi Wan Kenobi, you're my only hope");

or like this:

String help = "Help me, Obi Wan Kenobi, you're my only hope";

The above code that creates an instance of the String class should be familiar if you have experience creating Ruby classes or using Javascript Constructors.

Variables

We’ll end on something a tiny bit more interesting than just going over data types. Variables are a huge part of several coding languages, and Java is part of that club.

As usual, variables are used to store values. In Java however, each variable must be assigned a data type. Let’s go over a couple expamples of assigning variables.

int myAge = 30;
char myMathTestGrade = 'A';
boolean obiWanRocks = true;

The first word of each line assigns the data types (int, char or boolean), the second word names the variable (myAge, myMathTestGrade, obiWanRocks), the “=” sets what is on the left side of the =, equal to what is on the right side of the =.

Hopefully this makes it a little easier for some of you to stop avoiding Java and keep on learning!
